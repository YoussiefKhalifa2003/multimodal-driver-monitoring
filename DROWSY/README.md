# Module 2 — EEG Drowsiness Detection

Part of the [Multimodal Driver Monitoring System](../../README.md) · IEEE RAAI 2025

---

## What this module does

Processes a 10-channel EEG-style signal stream and classifies driver state as `drowsy` or `not_drowsy` using a Random Forest model. A real-time peak-detection layer monitors EEG amplitude spikes. When **5 or more drowsy events occur within a 10-second window**, a critical alert is triggered and an energetic Spotify playlist plays to wake the driver.

---

## Signal Processing Pipeline

```
Raw EEG stream (10 channels)
        │
        ▼
Sliding window (size = 50 samples)
        │
        ▼
Feature extraction per window:
  mean, std, max, min, mean-diff  ×  10 channels  =  50 features
        │
        ▼
StandardScaler (fit on training data, saved via joblib)
        │
        ▼
Random Forest Classifier
  (30 estimators, max_depth=3, max_features='sqrt')
        │
        ▼
Label: drowsy / not_drowsy
        │
        ▼
scipy.signal.find_peaks on real-time signal buffer
        │
        ▼
Critical alert if ≥5 events in last 10 seconds
```

---

## Model Details

| Parameter | Value |
|-----------|-------|
| Algorithm | Random Forest |
| Estimators | 30 |
| Max depth | 3 |
| Max features | `sqrt` |
| Min samples leaf | 5 |
| Input features | 50 (5 stats × 10 channels) |
| Serialization | `joblib` (`.pkl`) |

Intentionally shallow trees (low depth, few estimators) to avoid overconfident predictions on the synthetic dataset.

---

## Dataset

Synthetic EEG dataset generated by `generate_eeg_model.py`:
- 1,000 samples, 10 channels
- `drowsy` class: base Gaussian signal with periodic amplitude spikes (1.5–2.5×) to simulate abnormal EEG bursts
- `not_drowsy` class: steady Gaussian signal (σ=0.5)
- Saved to `synthetic_eeg_dataset.csv`

## Setup

```bash
cd "DROWSY/DROWSY"

python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # macOS / Linux

pip install -r requirements.txt

# Configure Spotify
cp ../../.env.example .env
# Edit .env with your Spotify credentials

# 1. Generate synthetic data and train the model
python generate_eeg_model.py    # creates synthetic_eeg_dataset.csv,
                                #   eeg_model_drowsy_binary.pkl,
                                #   eeg_scaler_drowsy_binary.pkl

# 2. Launch the live visualizer
python eeg_visualizer_balanced_fair.py
```

---

## Visualizer Controls

| Key | Action |
|-----|--------|
| `r` | Reset drowsy event counter and clear buffers |
| `m` | Regenerate model evaluation metrics |

The visualizer window shows:
- Live EEG signal with red peak markers
- Status overlay (`Normal` / `DROWSY - Peak Detected!`)
- Event counter: `Drowsy Events: N (Last 10s: N/5)`
- Critical alert overlay when threshold is breached

Evaluation metrics (confusion matrix, ROC curve, classification report) are auto-generated at startup and saved to `model_metrics/`.
