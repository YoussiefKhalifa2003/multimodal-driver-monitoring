## Overview

This system fuses three independent biosensing modalities — facial emotion recognition, EEG-based drowsiness detection, and EMG-based stress analysis — into a unified, closed-loop driver monitoring pipeline. A deterministic arbitration policy resolves conflicts across degraded or competing sensor streams to produce a stable driver-state estimate. The system reacts in real time by routing authenticated Spotify Web API calls to dynamically adjust in-car music, delivering **97% emotion-route accuracy** under the published validation methodology.

## Modules

### Module 1 — Facial Emotion Recognition (`FACE DETECT/`)

| Attribute | Detail |
|-----------|--------|
| **Model** | Custom CNN — 3 convolutional blocks (32→64→128 filters), BatchNorm, Dropout |
| **Input** | 64×64 grayscale face crops via Haar cascade |
| **Classes** | `happy` · `sad` · `angry` |
| **Dataset** | FER2013 (Kaggle) — download script included |
| **Output** | Emotion label + per-class probability overlay on live webcam feed |
| **Feedback** | Emotion-specific Spotify playlist triggered on stable detection (≥5 consistent frames) |
| **Key scripts** | `train_facial_emotion_model.py` · `fusion_driver_demo.py` · `evaluate_model.py` |

Training uses ImageDataGenerator augmentation (rotation, zoom, brightness), class-weight balancing, EarlyStopping, and ReduceLROnPlateau. A `SimulatedStateGenerator` provides a Markov-chain state model for testing without live sensor input.

---

### Module 2 — EEG Drowsiness Detection (`DROWSY/`)

| Attribute | Detail |
|-----------|--------|
| **Model** | Random Forest (30 estimators, max_depth=3, `sqrt` features) |
| **Input** | Synthetic 10-channel EEG — sliding window (size=50), 5 statistical features per channel |
| **Classes** | `drowsy` · `not_drowsy` |
| **Dataset** | Synthetic EEG generated by `generate_eeg_model.py` with realistic spike patterns |
| **Detection** | `scipy.signal.find_peaks` — triggers alert after 5+ drowsy events within 10 seconds |
| **Feedback** | Spotify energetic playlist; fallback pygame alert tone |
| **Key scripts** | `generate_eeg_model.py` · `eeg_visualizer_balanced_fair.py` · `retrain_model_enhanced.py` |

Features extracted per window: mean, std, max, min, mean-diff across all channels (50 features total). Scaler and model serialized via `joblib`.

---

### Module 3 — EMG Stress Analysis (`Stress/`)

| Attribute | Detail |
|-----------|--------|
| **Model** | MLP — Dense(64, relu) → Dropout(0.4) → Dense(32, relu) → Dropout(0.4) → Dense(1, sigmoid) |
| **Input** | 8-channel raw EMG from MYO Thalmic bracelet |
| **Classes** | Binary: `stressed` (1) · `not_stressed` (0) |
| **Dataset** | [EMG Gestures Dataset](https://www.kaggle.com/datasets/kyr7plus/emg-4) — 36 subjects, 72 files, ~MYO bracelet |
| **Phases** | Live simulation cycles through `not_stressed → stressed → excess_stress` |
| **Feedback** | Calm playlist on high stress ratio (≥50% of 60-sample window); regular playlist otherwise |
| **Key scripts** | `train_model.py` · `data_loader.py` · `live_simulation.py` · `graph.py` |

Labels derived from gesture class: rest/unmarked (0,1) → `not_stressed`; active gestures (2–7) → `stressed`. Model saved to `models/stress_model.h5` with scaler arrays.

## Quickstart

### Prerequisites
- Python 3.11
- A [Spotify Developer](https://developer.spotify.com/dashboard) account (free) and an active Spotify Premium device for playback control
- Webcam (Module 1)

### 1. Clone the repository
```bash
git clone https://github.com/<your-username>/multimodal-driver-monitoring.git
cd multimodal-driver-monitoring
```

### 2. Configure Spotify credentials
```bash
cp .env.example .env
# Open .env and fill in SPOTIFY_CLIENT_ID and SPOTIFY_CLIENT_SECRET
```

### 3. Set up each module

**Module 1 — Face Detection**
```bash
cd "FACE DETECT/FACE DETECT"
python -m venv .venv && .venv\Scripts\activate   # Windows
pip install -r requirements.txt
python download_fer2013.py          # Download FER2013 dataset
python train_facial_emotion_model.py
python fusion_driver_demo.py        # Run live demo
```

**Module 2 — EEG Drowsiness**
```bash
cd "DROWSY/DROWSY"
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
python generate_eeg_model.py        # Generate data + train Random Forest
python eeg_visualizer_balanced_fair.py
```

**Module 3 — EMG Stress**
```bash
cd "Stress/Stress"
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
python train_model.py               # Train MLP on EMG dataset
python live_simulation.py           # Run live stress simulation
```

| Module | Method | Reported Accuracy |
|--------|--------|-------------------|
| Facial Emotion | CNN (FER2013) | **97% emotion-route accuracy** |
| EEG Drowsiness | Random Forest + peak detection | Critical alert at 5+ events / 10 s |
| EMG Stress | MLP binary classifier | Validated on 36-subject MYO dataset |

Full validation methodology, confusion matrices, and ROC analysis are described in the published paper.

## Security Notice

Spotify API credentials (`client_id`, `client_secret`) must be stored in a `.env` file and are **never committed to this repository**. See `.env.example` for the required format. If you forked or cloned an earlier version of this codebase that contained raw credentials, rotate your Spotify app keys immediately at [developer.spotify.com/dashboard](https://developer.spotify.com/dashboard).
